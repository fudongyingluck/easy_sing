# 播放历史 — 红线交互设计文档

---

## 方案对比

| | 方案 A：纯红线 | 方案 B：进度条 + 画布浏览 |
|---|---|---|
| seek 方式 | 拖动红线 | 拖动进度条 |
| 波形浏览 | 拖动画布（纯视口） | 拖动画布（纯视口） |
| 红线贴边跟随 t_cursor | ✅ 需要 | ❌ 不需要 |
| 手势区分（红线 vs 画布） | ✅ 需要 | ❌ 不需要 |
| seek 精度 | 高 | 高 |
| 实现复杂度 | 高 | 低 |
| **当前选用** | | ✅ |

## 核心概念（两方案共用）

| 概念 | 含义 |
|------|------|
| `t_cursor` | 当前播放 / seek 位置（秒） |
| `viewport_start` | 视口左边缘对应的时刻（秒） |
| `seconds_per_screen` | 一屏显示多少秒，固定值（CONFIG.DEFAULT_CHART_DURATION） |
| 红线屏幕 x | `clamp((t_cursor - viewport_start) / seconds_per_screen * W, 0, W)` |

红线**始终可见**，clamp 到屏幕左右边缘，不会消失。

## 播放时 viewport 三段式（两方案共用）

```
示意（■ 表示红线位置）：
  阶段一：[■→      ]  视口固定在起点，红线持续从左向右移动，直到到达屏幕中央
  阶段二：[   →■←  ]  红线锁定在屏幕中央不动，视口持续向左滚动（画布向左推进）
  阶段三：[      →■]  视口到头停止滚动，红线继续从中央向右移动，最终停在右边缘

设 H = total_duration - seconds_per_screen / 2

阶段一：t_cursor ∈ [0, seconds_per_screen / 2]
  viewport_start = 0（视口固定）
  红线 x 持续右移：t_cursor / seconds_per_screen * W

阶段二：t_cursor ∈ [seconds_per_screen / 2, H]
  viewport_start = t_cursor - seconds_per_screen / 2（视口随时间滚动）
  红线始终在屏幕正中央（x = W / 2）

阶段三：t_cursor ∈ [H, total_duration]
  viewport_start = total_duration - seconds_per_screen（视口固定在末尾）
  红线 x 继续右移直到右边缘

特殊情况：total_duration <= seconds_per_screen
  viewport_start = 0，红线从左到右贯穿全程，不触发阶段二/三
```

## 方案 A：纯红线

### 暂停时手势区分

```
red_line_x = clamp((t_cursor - viewport_start) / seconds_per_screen * W, 0, W)

if |touch_x - red_line_x| <= HIT_SLOP (建议 20pt):
  → 红线拖动模式
else:
  → 画布拖动模式
```

### 画布拖动

```
delta（秒）= -dx_pixels * seconds_per_screen / W

viewport_start += delta

软边界（允许超出回弹）：
  [-bounce_margin, total_duration - seconds_per_screen + bounce_margin]
  bounce_margin = seconds_per_screen * 0.3

t_cursor 贴边跟随逻辑：
  x_raw = (t_cursor - viewport_start) / seconds_per_screen * W
  if x_raw < 0:
    t_cursor = max(0, viewport_start)          // 贴左边，t_cursor 跟着左边缘走
  if x_raw > W:
    t_cursor = min(total_duration, viewport_start + seconds_per_screen)  // 贴右边

松手回弹：Animated.spring → viewport_start 回到合法范围
```

### 红线拖动（seek）

```
拖动中：
  t_cursor = clamp(viewport_start + touch_x * seconds_per_screen / W, 0, total_duration)
  viewport_start 不变

松手后：onSeekChange(t_cursor)
```

## 方案 B：进度条 seek + 画布浏览（当前选用）

### 画布拖动（纯视口平移，不改变 t_cursor）

```
delta（秒）= -dx_pixels * seconds_per_screen / W

viewport_start += delta

软边界（允许超出回弹）：
  [-bounce_margin, total_duration - seconds_per_screen + bounce_margin]
  bounce_margin = seconds_per_screen * 0.3

t_cursor 不变

松手回弹：Animated.spring → viewport_start 回到合法范围
```

红线贴边时仅视觉贴边，t_cursor 不跟随。

### 进度条拖拽（seek）

```
拖动中：
  t_cursor = 拖拽比例 * total_duration
  audioService.seekTo(t_cursor)

松手后：
  viewport_start 重置为三段式对应值（视口跟到红线位置）
```

## 录音实时模式 — 红线三段式

录音时红线行为与播放时相同，`total_duration = recordingDurationLimit`（时长上限）。

```
示意（■ 表示红线位置）：
  阶段一：[■→      ]  视口固定在起点，红线持续从左向右移动，直到到达屏幕中央
  阶段二：[   →■←  ]  红线锁定在屏幕中央不动，视口持续向左滚动（画布向左推进）
  阶段三：[      →■]  视口到头停止滚动，红线继续从中央向右移动，最终停在右边缘

设 H = recordingDurationLimit - seconds_per_screen / 2

阶段一：currentTime ∈ [0, seconds_per_screen / 2]
  viewport 固定：startTime = 0，endTime = seconds_per_screen
  红线 x 位置 = currentTime / seconds_per_screen * W（持续右移）

阶段二：currentTime ∈ [seconds_per_screen / 2, H]
  viewport 随时间滚动：endTime = currentTime + seconds_per_screen / 2
  红线始终在屏幕正中央（x = W / 2）

阶段三：currentTime ∈ [H, recordingDurationLimit]
  viewport 固定在末尾：endTime = recordingDurationLimit
  红线 x 位置 = (currentTime - startTime) / seconds_per_screen * W（继续右移直到右边缘）

无时长上限时（recordingDurationLimit = 0）：
  只有阶段一和阶段二，没有阶段三
```

**视口公式**：
```
rawEnd   = max(seconds_per_screen, currentTime + seconds_per_screen / 2)
endTime  = recordingDurationLimit > 0 ? min(rawEnd, recordingDurationLimit) : rawEnd
startTime = max(0, endTime - seconds_per_screen)
```

**注意**：
- 到达时长上限后需将 `currentTime` 精确 clamp 到 `recordingDurationLimit`，
  避免定时器最后一次 tick 的浮点误差导致红线超出右边缘
- 涉及文件：`src/components/PitchChart.tsx`（视口计算）、`src/hooks/useRecording.ts`（clamp currentTime）
