# 播放历史 — 红线交互设计文档

---

## 方案对比

| | 方案 A：纯红线 | 方案 B：进度条 + 画布浏览 |
|---|---|---|
| seek 方式 | 拖动红线 | 拖动进度条 |
| 波形浏览 | 拖动画布（纯视口） | 拖动画布（纯视口） |
| 红线贴边跟随 t_cursor | ✅ 需要 | ❌ 不需要 |
| 手势区分（红线 vs 画布） | ✅ 需要 | ❌ 不需要 |
| seek 精度 | 高 | 高 |
| 实现复杂度 | 高 | 低 |
| **当前选用** | | ✅ |

---

## 核心概念（两方案共用）

| 概念 | 含义 |
|------|------|
| `t_cursor` | 当前播放 / seek 位置（秒） |
| `viewport_start` | 视口左边缘对应的时刻（秒） |
| `seconds_per_screen` | 一屏显示多少秒，固定值（CONFIG.DEFAULT_CHART_DURATION） |
| 红线屏幕 x | `clamp((t_cursor - viewport_start) / seconds_per_screen * W, 0, W)` |

红线**始终可见**，clamp 到屏幕左右边缘，不会消失。

---

## 播放时 viewport 三段式（两方案共用）

```
设 H = total_duration - seconds_per_screen / 2

阶段一：t_cursor ∈ [0, seconds_per_screen / 2]
  viewport_start = 0
  红线从左边缘向右移动至屏幕中央

阶段二：t_cursor ∈ [seconds_per_screen / 2, H]
  viewport_start = t_cursor - seconds_per_screen / 2
  红线固定在屏幕中央，画布向左滚动

阶段三：t_cursor ∈ [H, total_duration]
  viewport_start = total_duration - seconds_per_screen（画布到头，停止）
  红线从屏幕中央继续向右移动至右边缘

特殊情况：total_duration <= seconds_per_screen
  viewport_start = 0，红线从左到右，不触发阶段二/三
```

---

## 方案 A：纯红线

### 暂停时手势区分

```
red_line_x = clamp((t_cursor - viewport_start) / seconds_per_screen * W, 0, W)

if |touch_x - red_line_x| <= HIT_SLOP (建议 20pt):
  → 红线拖动模式
else:
  → 画布拖动模式
```

### 画布拖动

```
delta（秒）= -dx_pixels * seconds_per_screen / W

viewport_start += delta

软边界（允许超出回弹）：
  [-bounce_margin, total_duration - seconds_per_screen + bounce_margin]
  bounce_margin = seconds_per_screen * 0.3

t_cursor 贴边跟随逻辑：
  x_raw = (t_cursor - viewport_start) / seconds_per_screen * W
  if x_raw < 0:
    t_cursor = max(0, viewport_start)          // 贴左边，t_cursor 跟着左边缘走
  if x_raw > W:
    t_cursor = min(total_duration, viewport_start + seconds_per_screen)  // 贴右边

松手回弹：Animated.spring → viewport_start 回到合法范围
```

### 红线拖动（seek）

```
拖动中：
  t_cursor = clamp(viewport_start + touch_x * seconds_per_screen / W, 0, total_duration)
  viewport_start 不变

松手后：onSeekChange(t_cursor)
```

## 方案 B：进度条 seek + 画布浏览（当前选用）

### 画布拖动（纯视口平移，不改变 t_cursor）

```
delta（秒）= -dx_pixels * seconds_per_screen / W

viewport_start += delta

软边界（允许超出回弹）：
  [-bounce_margin, total_duration - seconds_per_screen + bounce_margin]
  bounce_margin = seconds_per_screen * 0.3

t_cursor 不变

松手回弹：Animated.spring → viewport_start 回到合法范围
```

红线贴边时仅视觉贴边，t_cursor 不跟随。

### 进度条拖拽（seek）

```
拖动中：
  t_cursor = 拖拽比例 * total_duration
  audioService.seekTo(t_cursor)

松手后：
  viewport_start 重置为三段式对应值（视口跟到红线位置）
```

## 组件 Props（两方案共用）

```ts
interface PlaybackPitchChartProps {
  data: PitchDataPoint[]
  minNote: string
  maxNote: string
  totalDuration: number
  currentTime: number            // 播放位置，持续变化
  isPlaying: boolean             // true 时禁用手势
  height?: number
  onSeekChange?: (time: number) => void  // 方案 A 红线拖动时回调
}
```

### 内部状态

```ts
viewportStart: number           // 当前视口左边缘时刻
// 方案 A 额外需要：
gestureMode: 'none' | 'canvas' | 'redline'
panStartCursor: number
// 两方案共用：
panStartViewport: number
bounceAnim: Animated.Value
```
