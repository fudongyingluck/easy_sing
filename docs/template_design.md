# 音高模板功能设计文档

## 功能目标

允许用户导入一段音频，分析其音高曲线作为「参考模板」。练习时选择模板，模板音高以半透明曲线显示在图表背景层，实时演唱的音高显示在前层，辅助用户对准音高。

---

## 核心概念

- **模板（PitchTemplate）**：可作为练习参考的音高数据，来源有两种：导入的外部音频、或直接从录音转换
- **参考曲线**：练习时叠加在图表背景的半透明音高线，来自模板
- **实时曲线**：演唱时的实时音高，叠加在参考曲线之上

---

## 模板来源

点击模板页右上角「＋」后，弹出来源选择 ActionSheet，包含以下选项：

### 来源一：从文件导入
- 调用系统 `UIDocumentPickerViewController`
- 支持格式：mp3、m4a、wav、aac
- 选中后返回原始文件路径（不立即复制到沙盒）
- JS 层读取音频时长，若超过 10 分钟：
  - 弹提示：「该音频时长超过 10 分钟，仅支持导入前 10 分钟，是否继续？」
  - 用户取消 → 放弃导入，不做任何操作（无需清理，文件未复制）
  - 用户确认 → 进入下一步
- 用户确认后，将文件复制到沙盒目录 `Documents/PitchPerfect/Imports/`，再触发音高分析
- 需要分析音高（后台跑 YIN，显示 Loading）

### 来源二：从微信导入
- 点击后直接通过 URL Scheme（`weixin://`）跳转到微信
- 同时在页面显示操作引导文字：「请在微信中找到音频文件，长按 → 用其他应用打开 → 选择本 app」
- 用户完成操作后，微信通过 iOS Share Sheet 将文件回传给本 app，app 自动接收并进入分析流程
- 需在 `Info.plist` 注册 `CFBundleDocumentTypes`，声明支持 mp3 / m4a / wav / aac
- 需在 `LSApplicationQueriesSchemes` 中注册 `weixin`，才能检测微信是否安装
- 若微信未安装，按钮置灰并提示「未检测到微信」
- **注意**：微信语音消息为 `.silk` 格式，iOS 无法解码，不支持导入

### 来源三：从录音历史选择
- 点击后在模板页内展示录音历史列表，用户选择一条
- **无需重新分析**，音频文件和音高数据直接引用，不复制文件
- 模板记录中保存 `sourceRecordingId` 字段，标明来源录音

## 音高分析

在 `PitchDetectorModule.mm` 中新增 `analyzeAudioFile` 方法：

- 使用 `AVAudioFile` 读取并自动解码为 float32 PCM（支持所有上述格式）
- 在后台线程执行，不阻塞 UI
- 复用现有 YIN 算法实现
- **YIN 检测频率**：内部可按文件采样率自然步进（无需节流）
- **存储采样率：10 Hz**（每 100ms 存一个音高点，与录音侧 `PITCH_DATA_SAMPLE_RATE = 10` 一致；录音侧实际检测是 100 Hz，但存储时节流到 10 Hz）
- 频率有效范围：60 Hz – 1400 Hz（超出范围记为 0）
- 最大支持时长：10 分钟
- 返回格式：`{ points: [{time, freq}], duration }`

## 数据模型

```typescript
interface PitchTemplate {
  id: string                      // UUID
  name: string                    // 用户可编辑的名称
  sourceFileName: string          // 原始文件名，仅展示用
  audioFilePath: string           // 沙盒内文件名（不含目录）
  pitchDataKey: string            // AsyncStorage key，格式同录音
  duration: number                // 秒
  createTime: string              // ISO 日期字符串
  sourceRecordingId?: string      // 若来源为录音转换，记录原录音 id
}
```

- 音高数据格式与现有 `PitchData` 完全一致，复用现有存储逻辑
- 从录音转换时，`audioFilePath` 和 `pitchDataKey` 直接指向原录音的数据，不复制文件

---

## 存储

| 内容 | 位置 |
|------|------|
| 模板元数据 | AsyncStorage，key：`@pitchperfect:templates` |
| 音高数据（导入来源） | AsyncStorage，key：`@pitchperfect:pitchdata:template_<id>` |
| 音高数据（录音来源） | 直接复用录音的 `pitchDataKey`，不新建 |
| 音频文件（导入来源） | `Documents/PitchPerfect/Imports/<filename>` |
| 音频文件（录音来源） | 直接复用录音的 `audioFilePath`，不复制 |

### 删除联动规则
- **删除模板** → 只删模板元数据；若来源是导入文件则同时删音频文件和音高数据；若来源是录音则不动原录音
- **删除录音** → 若该录音被某模板引用（`sourceRecordingId` 匹配），弹二次确认：「该录音已被设为模板"XXX"，是否同时删除该模板？」
  - 选「同时删除」→ 录音和模板一并删除，练习页清空选中状态
  - 选「保留模板」→ 先将录音的音频文件复制一份到 `Imports/` 目录，更新模板的 `audioFilePath` 指向副本，再删除原录音；模板独立存活，不受影响

## 界面设计

### 模板 Tab（TemplatesScreen）
模板列表与录音列表功能对齐，差别仅在于多了「在练习中使用」的能力。

- 模板列表：名称、时长、来源（文件名 or 录音名）
- 右上角「＋」按钮：触发导入流程
- 导入流程：选择文件 → 分析中显示 Loading → 完成后插入列表
- **点击单条**：进入播放页（与录音播放页一致，可回放音频 + 查看音高曲线）
- 每行常驻显示「分享（↗）」「删除（✕）」按钮（与录音列表一致）
- Header 右上角「选择」按钮：进入多选模式
  - 列表每行显示复选框，常驻按钮隐藏
  - Header 变为「取消」+「全选/全不选」
  - 已选数量显示在标题处
  - 底部出现「删除」按钮，点击后弹确认弹窗，批量删除
- **重命名**：长按名称文字弹 Alert 输入框（与录音列表一致）

### 录音列表（RecordingsScreen）
- 无需改动，模板导入入口统一在模板页

### 练习页（PracticeScreen）
- Header 右侧始终显示一个文字按钮（无图标，样式与录音列表的「选择」按钮一致）：
  - 未选中时显示「无模板」
  - 选中后显示模板名称前 4 个字符（超出截断）
- 点击后始终弹出 Modal：
  - **有模板时**：「不使用模板」选项 + 模板列表
  - **无模板时**：空状态提示文字 + 「前往模板页导入」按钮（关闭 Modal 并跳转模板 Tab）
- **已选模板被删除后**：练习页自动清空选中状态，按钮恢复显示「无模板」

## 可视化叠加规则

| 层级 | 内容 | 颜色 | 透明度 | 线宽 |
|------|------|------|--------|------|
| 底层 | 模板参考曲线 | `#FF9500`（橙色） | 0.25 | 2.5px |
| 前层 | 实时演唱曲线 | `#007AFF`（蓝色） | 1.0 | 2.5px |

- 时间轴对齐：模板和实时录音均从 t=0 开始，共用同一时间轴
- 若实时录音时长超过模板，模板曲线自然结束，不循环
- 模板曲线使用与实时曲线相同的 Catmull-Rom 插值渲染

---

## 约束与限制

- 仅支持 iOS，Android 暂不处理
- 微信语音消息（.silk）不支持
- 不支持时间偏移对齐（即不能选择从模板第 N 秒开始匹配）
- 不需要新增 Xcode 源文件，新方法加入现有模块文件

## 模板音频同步播放

选中模板后，练习时可同时播放模板原始音频作为参考，用户跟着唱。

### 耳机检测逻辑

| 状态 | 行为 |
|------|------|
| 已连接耳机 / 蓝牙音频 | 直接播放，无提示 |
| 未连接耳机 | 弹提示：「未检测到耳机，建议戴耳机以获得最佳效果。」用户确认后继续播放 |

### 回声消除

- 无耳机时，模板音频从扬声器播出会被麦克风收入
- 现有代码已启用系统 AUVoiceIO 语音处理（`setVoiceProcessingEnabled:YES`），包含回声消除（AEC）
- AEC 会将扬声器播出的音频从麦克风信号中消除，保留人声，**无需额外开发**

### 播放控制

- 开始录音时自动开始播放模板音频（从头开始）
- 暂停录音时同步暂停模板音频
- 继续录音时同步恢复
- 停止/放弃录音时停止模板音频
- 模板音频播放完毕后静默结束，录音继续

### 耳机检测方式

通过 `AVAudioSession.currentRoute.outputs` 判断当前音频输出端口，若包含 `headphones`、`bluetoothA2DP`、`bluetoothHFP` 则认为已连接耳机。
