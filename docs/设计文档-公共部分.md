# 实时音准练习 - 公共设计文档

## 项目概述

### 产品名称
实时音准练习

### 产品定位
帮助唱歌爱好者和声乐练习者通过实时音高检测和虚拟钢琴辅助提高音准的应用。

### 核心价值
- 实时反馈音高，直观展示音准偏差
- 按男/女声音域优化，更贴合实际使用
- 纯工具类应用，无版权风险

---

## 功能需求

### 1. 音域模式切换

#### 模式列表
| 类型 | 模式 | 音域范围 | 频率范围 | 适用人群 |
|------|------|----------|----------|----------|
| 预设 | ♂ 男声模式 | E2 - C5 | 80Hz - 520Hz | 男生（含男低/中/高音） |
| 预设 | ♀ 女声模式 | C3 - C6 | 160Hz - 1040Hz | 女生（含女中/高音） |
| 预设 | 🎸 吉他模式 | E2 - E6 | 82Hz - 1319Hz | 吉他练习 |
| 预设 | 🎻 小提琴模式 | G3 - E7 | 196Hz - 2637Hz | 小提琴练习 |
| 自定义 | 用户命名 | 用户设定 | 用户设定 | 用户保存的自定义模式 |

*注：可根据需求随时扩展更多预设模式（如男低音、男高音、女低音、花腔女高音、钢琴等）*

#### 模式切换功能
- 模式选择使用下拉列表形式，位于界面顶部
- 显示当前选中模式名称
- 点击展开下拉列表，可选择所有预设模式和用户保存的自定义模式
- 切换模式时，音高检测范围和钢琴显示范围同步更新
- 如果正在录音，提示"切换模式将停止录音"，确认后停止
- **自动记忆**：应用启动时自动加载用户上次使用的模式

#### 自定义模式管理
- **创建自定义模式**：
  - 可选择起始音和结束音
  - 输入模式名称（如"我的高音区"、"男低音练习"）
  - 提供钢琴预览
  - 保存到本地存储

- **自定义模式列表**：
  - 显示所有用户保存的自定义模式
  - 支持编辑已保存的自定义模式
  - 支持删除不需要的自定义模式
  - 最多保存 20 个自定义模式

- **模式命名规则**：
  - 名称长度：1-20个字符
  - 支持中文、英文、数字、 emoji
  - 名称不能重复

### 2. 录音与钢琴模式切换

#### 两种模式定义

| 模式 | 说明 | 可用功能 |
|------|------|----------|
| **录音模式**（默认） | 专注于录音和音高检测 | 录音、实时音高检测、音高曲线显示 |
| **钢琴模式** | 专注于钢琴弹奏和听参考音 | 钢琴点击发音、无音高检测 |

#### 模式切换交互

- **切换触发**：双击"音域区"（显示"E2 ~ C5"的区域）
- **切换方向**：在录音模式和钢琴模式之间切换
- **录音状态处理**：
  - 从录音模式切换到钢琴模式：录音暂停（不停止）
  - 从钢琴模式切换回录音模式：录音继续（从暂停处恢复）

#### 模式状态说明

**录音模式：**
- 虚拟钢琴禁用/变灰
- 显示提示："双击音域区切换到钢琴模式"
- 录音和音高检测功能正常

**钢琴模式：**
- 虚拟钢琴可用，可点击发音
- 不显示音高曲线
- 不进行音高检测
- 显示提示："双击音域区切回录音模式"
- 录音处于暂停状态

### 3. 实时音高检测

#### 技术要求
- **实时性**：延迟 < 50ms
- **准确性**：能够准确识别唱歌时的音高
- **抗噪性**：对环境噪音有一定抗干扰能力

#### 技术方案
- 使用平台原生音频处理 API
- 实时流式处理
- YIN 算法进行音高检测
- 采样率：22050Hz（兼顾性能和精度）
- 分析窗口：2048点

---

### 4. 虚拟钢琴

#### 钢琴特性
- **范围同步**：钢琴显示范围与当前音高检测范围完全一致
- **可折叠**：可向上展开/向下收起
- **点击发音**：点击琴键播放标准音高（正弦波）
- （注：检测到音高时不需要高亮钢琴，只在曲线图上展示）

#### 钢琴配置
- 白键宽度：根据屏幕宽度自适应
- 黑键宽度：约为白键的 60%
- 中央C定位：提供一键定位到 C4 的功能

#### 钢琴尺寸
- 展开高度：手机屏幕高度的 1/3（自适应）
- 收起状态：只显示标题和三角形箭头，不显示琴键
- 收起提示：显示"▼ 虚拟钢琴"或"▲ 虚拟钢琴"

#### 琴键配置
- 白键最小宽度：36px（保证可点击）
- 钢琴范围较长时：支持左右滑动
- 滑动指示器：顶部显示滑块，展示当前可见区域位置
- 默认显示：根据音域范围自动定位到中间部分

#### 滑动边界处理
- 滑到最左侧或最右侧时：不继续响应滑动，直接停在边界
- 不做回弹效果或闪动提示

#### 滑块与钢琴滚动同步
- 滑块位置与钢琴滚动位置双向绑定
- 拖动滑块时，钢琴同步滚动到对应位置
- 滑动钢琴时，滑块同步移动到对应位置
- 滑块比例 = 钢琴可见区域宽度 / 钢琴总宽度

#### 钢琴交互
- 点击琴键：播放标准音高（正弦波）
- （注：检测到音高时不需要高亮钢琴键）

#### 定位中央C
- 不需要单独的"定位中央C"按钮
- 根据当前音域范围，自动显示中间部分
- 中央C（C4）在音域范围内时，优先让C4可见

### 5. 录音功能

#### 录音参数
- **最大时长**：10分钟
- **格式**：mp3
- **采样率**：44100Hz
- **声道**：单声道
- **编码码率**：96kbps

#### 录音流程
1. 点击"开始录音"按钮
2. 同时开始实时音高检测和显示
3. 显示录音时长
4. 点击"停止录音"或达到最大时长自动停止
5. 停止后显示操作选项：试听、保存、放弃

#### 录音时的显示
- 开始录音时，在曲线图上显示时间轴标记
- 时间轴标记与录音进度同步

#### 试听录音
- 播放录音时，曲线图同步回放音高曲线
- 播放进度与曲线位置对应

#### 音高数据保存
- 录音时同步保存音高数据，试听时直接读取，不重新分析
- 保存格式：JSON 文件
- 数据采样率：每秒 10 个数据点（可配置）
- 数据结构：
  ```javascript
  {
    version: 1,
    sampleRate: 10,      // 每秒10个数据点
    duration: 155,       // 总时长(秒)
    data: [
      { time: 0.0, freq: 261.63, note: 'C4' },
      { time: 0.1, freq: 0, note: null },
      // ...
    ]
  }
  ```

#### 文件存储方案
- 录音文件：`rec_{id}.mp3`
- 音高数据文件：`rec_{id}_pitch.json`
- 录音列表记录中同时保存两个文件的路径
- 删除录音时，同时删除两个文件

#### 临时文件管理
- 停止录音后，临时文件保留到退出应用
- 退出应用时自动删除临时录音文件
- 未保存的录音不会保留

#### 文件命名
- 命名格式：`YYYY-MM-DD HH:mm:ss`（带秒防止重复）
- 重命名规则：
  - 长度限制：20个字符以内
  - 特殊字符过滤：只允许中文、英文、数字、空格、下划线
  - 重复处理：重名时自动在后面加 `_n`（如 `我的录音_1`）

---

### 6. 录音管理

#### 录音列表
- **排序**：按时间倒序（最新在最上面）
- **显示内容**：
  - 录音名称（时间命名）
  - 录音时长
  - 创建时间
- **操作**：
  - 播放
  - 重命名
  - 删除

#### 录音命名
- 默认命名：`YYYY-MM-DD HH:mm:ss`
- 支持用户手动重命名

---

### 7. 退出保存逻辑

#### 退出场景
- 点击左上角返回按钮
- 点击系统返回键
- 切换到其他页面

#### 保存询问
- 有未保存的录音时，弹出确认对话框
- 对话框选项：[不保存] [保存]
- **默认选中**：不保存
- 保存内容：录音文件

## 核心算法

### YIN 音高检测算法

YIN 是一种专门为语音和音乐设计的音高检测算法，具有以下优点：
- 精度高
- 抗噪性好
- 适合实时处理

#### 算法步骤
1. 自相关函数计算
2. 差值函数计算
3. 归一化
4. 阈值选择
5. 峰值检测
6. 抛物线插值

#### 参数配置
```javascript
const YIN_CONFIG = {
  threshold: 0.15,           // YIN 阈值
  sampleRate: 22050,         // 采样率
  bufferSize: 2048,          // 分析窗口大小
  overlap: 0.5,              // 重叠比例
  minFreq: 80,               // 最小频率
  maxFreq: 1200,             // 最大频率
  confidenceThreshold: 0.85   // 置信度阈值
}
```

### 频率与音符转换

#### 频率计算公式
```
频率 = 440 * 2^((n - 69)/12)
其中 n = MIDI 音符编号（C4 = 60）
```

#### 音符名称映射
```javascript
const NOTE_NAMES = ['C', 'C#', 'D', 'D#', 'E', 'F', 'F#', 'G', 'G#', 'A', 'A#', 'B']
```

## 详细设计说明

### 1. 音高曲线图细节

#### X轴（时间轴）
- 显示范围：最近 5 秒（可配置变量，默认 5 秒）
- 时间标记：显示录音时间，如 "1:30" 表示 1 分 30 秒
- 起始点：左侧为当前时间减去显示范围

#### Y轴（音符轴）
- 标注方式：按全音音阶标注带八度数字的音符名，如 C4, D4, E4, F4, G4, A4, B4
- 标注规则：全全半全全全半（CDEFGAB）
- 根据当前模式的音域范围自动调整标注的起始和结束音符
- 只标注白键音符（C/D/E/F/G/A/B），不标注黑键

#### 曲线样式
- 绘制方式：连续的线
- 颜色：主色调（如蓝色）
- 线条宽度：2px

#### 无效音高处理
- 检测不到有效音高时，曲线中断（不显示）
- 此逻辑由 YIN 算法库处理，置信度低于阈值时不输出

---

### 2. 模式切换细节

#### 音域模式切换（预设/自定义）
- 正在录音时，点击音域模式按钮（男声/女声等），提示"切换模式将停止录音"
- 用户确认后停止录音，切换模式

#### 自定义模式创建
- 起始音校验：起始音不能大于结束音
- 如果起始音 > 结束音，提示"起始音必须小于结束音"

#### 删除自定义模式
- 如果删除的是当前正在使用的模式，自动切换到列表第一个模式
- 优先切换到预设模式，没有预设模式时切换到第一个自定义模式

#### 预设模式保护
- 预设模式（男声、女声、吉他、小提琴）不能编辑或删除
- 只能编辑和删除用户创建的自定义模式

---

### 3. 音高检测细节

#### 无效音高显示
- 检测不到有效音高时，不显示任何内容（不显示"--"）

#### 置信度处理
- 置信度只在内部使用，不显示给用户
- 置信度低于阈值时，不输出音高数据

#### 音准判定
- 准确阈值：设置为全局常量 `PITCH_ACCURACY_THRESHOLD`
- 默认值：±10 音分（可体验后调整）
- 偏差在阈值范围内算"准确"

---

### 4. 存储空间管理细节

#### 存储空间清理
- 超过 8MB 时，提示用户清理，不自动删除
- 清理提示：显示"存储空间不足，请清理旧录音"

#### 录音文件限制
- 没有单条文件大小限制
- 只有最长时长限制：10 分钟

#### 空间不足处理
- 提示用户清理，不自动删除
- 清理对话框：
  - 默认勾选：10 天以前的录音
  - 10 天设置为全局常量 `AUTO_CLEAN_DAYS`，方便修改
  - 用户可以手动选择要删除的录音

---

### 5. 页面跳转细节

#### 录音列表返回
- 从录音列表返回主界面时，如果有未保存的录音，弹出确认对话框
- 选项：[不保存并返回] [保存并返回]

#### 自定义模式设置页面
- 左上角按钮：返回
- 未保存的修改：直接丢弃，不提示保存
- 用户点击"保存"才会保存修改

### 双击检测实现

应用不直接支持双击事件，需要自己实现：

```javascript
let lastTapTime = 0

function handleDoubleTap(callback) {
  const currentTime = Date.now()
  const timeDiff = currentTime - lastTapTime

  if (timeDiff < CONFIG.DOUBLE_TAP_DELAY && timeDiff > 0) {
    // 检测到双击
    callback()
    lastTapTime = 0
  } else {
    // 记录点击时间
    lastTapTime = currentTime
  }
}
```
